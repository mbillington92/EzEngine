# EzEngine.ContentManagement

This library is intended strictly to load PolyOne files with little to no further processing or consideration for its contents. 
You're highly recommended to seek out an additional library for whatever framework or engine you're using to simplify your usage of PolyOne files by converting their contents into a more useful complex object structure. For an example of this, take a look at EzEngine.ContentManagement.Mono.Interop. However, if you're a more advanced user or want to use PolyOne files in highly specific way, this library provides a way to do so.

## Dependencies

- .NET 8
- ServiceStack.Text - [text](https://github.com/ServiceStack/ServiceStack.Text?tab=readme-ov-file)
